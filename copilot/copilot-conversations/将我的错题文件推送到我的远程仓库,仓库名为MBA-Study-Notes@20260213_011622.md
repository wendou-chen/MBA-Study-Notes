**ai**: 在 Obsidian 里用 **Obsidian Git** 插件设置远程仓库地址，本质上就是给你的 vault 对应的 Git 仓库设置 `remote`（通常叫 `origin`）。按下面做即可。

## 1）先确认你的 vault 已经是一个 Git 仓库
在 Obsidian 中：
- `Ctrl/Cmd + P` 打开命令面板
- 搜索并运行：`Obsidian Git: Open repository in terminal`（或类似命令名）
- 在弹出的终端里执行：

```bash
git status
```

如果显示 “not a git repository”，说明这一步还没初始化，需要在 vault 根目录执行：

```bash
git init
```

## 2）设置/添加远程仓库地址（origin）
还是在 vault 根目录终端执行：

### 情况 A：你还没有远程仓库
```bash
git remote -v
```
如果没有输出（或没有 `origin`），添加远程：

- HTTPS（最常用）：
```bash
git remote add origin https://github.com/<你的用户名>/MBA-Study-Notes.git
```

- SSH（你配置过 SSH key 才用这个）：
```bash
git remote add origin git@github.com:<你的用户名>/MBA-Study-Notes.git
```

然后检查：
```bash
git remote -v
```

### 情况 B：已经有 origin，但地址不对/想改
```bash
git remote set-url origin https://github.com/<你的用户名>/MBA-Study-Notes.git
```

检查：
```bash
git remote -v
```

## 3）在 Obsidian Git 插件里确认推送分支（避免 push 失败）
Obsidian → Settings → (Community plugins) → **Obsidian Git**，重点看：
- Push/Pull 的默认分支：一般是 `main`（有的仓库是 `master`）

你也可以用命令行确认当前分支名：
```bash
git branch
```

## 4）第一次推送（可选，但通常需要）
终端执行：

```bash
git add .
git commit -m "Initial commit"
git push -u origin main
```

如果你的分支是 `master` 就把 `main` 换成 `master`。
