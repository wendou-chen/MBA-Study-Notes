# 抖音素材排布规范

创建抖音图文素材文件时，必须在每个内容区域预留足够的空白行，让用户可以直接在对应位置粘贴图片或输入文案，无需手动创建空行。

## 核心原则

1. **图片区**：callout 提示后留 **5 行空行**，用户直接在空白处 Ctrl+V 粘贴
2. **文案区**：callout 提示后留 **5 行空行**，用户直接在空白处输入文字
3. **已有图片之间**：每张图片嵌入之间必须有 **1 行空行**
4. **节与节之间**：每个 `##` 标题前必须有 **2 行空行**（视觉分隔）

## 标准模板

```markdown
---
date: {date}
type: douyin-draft
status: 待发布
style: ""
tags:
  - 抖音
  - 素材
---

# 📱 {short_date} 抖音图文素材


## 📸 图片素材

> [!info] 在下方空白处粘贴图片（Ctrl+V）
> 抖音图文上限 **9 张**，建议 3–6 张。图片会自动保存到 `images/` 子目录。





## ✏️ 文案草稿

> [!tip] 在下方空白处写下你想表达的内容
> 关键词、情绪、灵感片段都行，AI 会帮你润色成完整文案。






## 🏷️ 标签备选

#考研 #27考研 #备考日常 #考研日记 #考研数学


## 📋 发布检查

- [ ] 图片已粘贴（≤ 9 张）
- [ ] 文案草稿已填写
- [ ] 风格已选择（任务汇报 / 情感润色）
- [ ] 准备发布 → 告诉 AI「发抖音」
```

## 排布规则详解

### ✅ 正确示范

```markdown
## 📸 图片素材

> [!info] 在下方空白处粘贴图片（Ctrl+V）
> 抖音图文上限 **9 张**，建议 3–6 张。

![[photo1.png]]

![[photo2.png]]

![[photo3.png]]


## ✏️ 文案草稿
```

- callout 后 5 行空行（粘贴区）
- 图片之间 1 行空行（防止粘连）
- 下一个 `##` 前 2 行空行（视觉分隔）

### ❌ 错误示范

```markdown
## 📸 图片素材
> [!info] 粘贴图片
![[photo1.png]]![[photo2.png]]
## ✏️ 文案草稿
```

问题：
- callout 后无空行 → 用户无处粘贴
- 图片粘连在同一行 → Obsidian 渲染异常
- 节之间无空行 → 视觉混乱

## 适用场景

- `douyin-publisher` skill 调用 `ensure_daily_note()` 创建新素材文件时
- AI 手动创建抖音素材 Markdown 文件时
- 用户说「创建今日素材」「新建抖音素材」时

## 与 douyin-publisher 的关系

本 skill 是 `douyin-publisher` 的排布子规范。`douyin_publish.py` 中的 `DAILY_NOTE_TEMPLATE` 必须遵循本规范的模板格式。当两者冲突时，以本 skill 为准。
