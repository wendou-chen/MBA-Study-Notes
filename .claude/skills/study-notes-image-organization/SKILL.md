# 错题笔记图片归档

保持仓库根目录整洁，让每个错题章节“笔记 + 图片资源”放在同一目录体系内，便于迁移、同步与复习。

## 标准流程

1. 定位目标章节
   - 找到章节笔记（例如：`考研数学/错题/<章节>/<章节>错题.md`）。
   - 识别该笔记里嵌入的图片引用（通常是 `![[...]]`）。

2. 创建章节图片目录
   - 在章节目录下创建 `images/`（例如：`考研数学/错题/微分方程/images/`）。

3. 移动图片文件
   - 已被 Git 跟踪的图片：优先用 `git mv`（保留历史）。
   - 未跟踪的图片：先移动，再 `git add`。

4. 更新 Obsidian 图片嵌入引用
   - 当 `images/` 与笔记同级时，推荐写法：`![[images/<filename>.png]]`。
   - 常见批量修复（图片原先在根目录）：将 `![[Pasted image` 替换为 `![[images/Pasted image`。

5. 校验
   - 根目录不再残留本章节的 `Pasted image ...`。
   - 每个 `![[images/...]]` 都能在文件系统中找到对应图片。

6. 可选清理（仅在明确安全时）
   - 删除明显无引用且内容为空壳的 `未命名*.base`。
   - 提交前用 `git status` 检查，避免把 `.env`、`.obsidian/`、`.opencode/`、`logs/` 等本地/敏感目录误提交。

## OpenCode 操作要点

- 用 `glob` 找章节笔记与潜在图片文件。
- 用 `grep` 追踪某张图片被哪些笔记引用。
- 用 Git 精准暂存（只 add 本次整理相关文件），避免混入无关改动。
