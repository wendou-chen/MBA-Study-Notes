# 考研每日计划 Skill

创建 `考研计划/` 目录下的每日计划文件时，必须严格遵循以下规范，否则 kaoyan-countdown 插件无法解析任务。

## 文件命名

```
考研计划/YYYY-MM-DD 周X.md
```

示例：`考研计划/2026-02-18 周三.md`

插件通过日期前缀 `${planFolder}/${YYYY-MM-DD}` 匹配文件，文件名必须以日期开头。

## Frontmatter（必须）

```yaml
---
date: 2026-02-18
weekday: 周三
phase: 1
type: daily-plan
status: pending
tags: [daily-plan, phase-1]
---
```

- `status`: 当日未结束用 `pending`，已复盘用 `completed`

## 任务格式（关键）

插件 `dailyParser.ts` 使用以下正则解析任务：

```
CHECKLIST_RE = /^- \[([ xX])\]\s*(.+?)\s*\|\s*(.+?)\s*\|\s*(.+)/
```

### 必须使用三段式格式

```markdown
- [ ] 时间 | 科目 | 描述
- [x] 时间 | 科目 | 描述
```

三个字段用 `|` 分隔，缺一不可：
1. **时间**：具体时段（如 `07:30 – 08:30`）或模糊时段（如 `上午`、`全天`、`—`）
2. **科目**：带 emoji 的科目标签（如 `🔢 数学`、`🔤 英语`、`📡 专业课`、`💻 项目`）
3. **描述**：任务具体内容

### 正确示例

```markdown
- [ ] 07:30 – 08:30 | 🔤 英语 | 墨墨背单词 **150 个**
- [ ] 08:30 – 10:30 | 🔢 数学 | 880 基础篇 · 微分方程 **10 题**
- [x] 上午 | 🔢 数学 | 微分方程刷题约 **10 题**
- [ ] — | 📡 专业课 | 未推进
```

### 错误示例（插件无法解析）

```markdown
- [x] 🔤 英语 | 墨墨背单词打卡          ← 缺少时间字段，只有两段
- [x] 墨墨背单词打卡                      ← 完全没有 | 分隔
- [x] 数学 | 上午 | 刷题 10 道            ← 科目和时间顺序反了
```

## 插件也支持表格格式（备选）

```
TABLE_ROW_RE = /^\|\s*(.+?)\s*\|\s*(.+?)\s*\|\s*(.+?)\s*\|(⬜|✅)\s*\|/
```

```markdown
| 时间 | 科目 | 内容 | 状态 |
|:----:|:----:|:----:|:----:|
| 07:30 – 08:30 | 🔤 英语 | 墨墨背单词 150 个 | ⬜ |
| 08:30 – 10:30 | 🔢 数学 | 880 微分方程 10 题 | ✅ |
```

## 整体模板

```markdown
---
date: YYYY-MM-DD
weekday: 周X
phase: 1
type: daily-plan
status: pending
tags: [daily-plan, phase-1]
---

# 📋 M.DD 周X · 主题标题

> [!tip] 今日战略重心
> **一句话概括今日核心目标。**

---

## 🕐 时间表

- [ ] 07:30 – 08:30 | 🔤 英语 | 具体任务描述
- [ ] 08:30 – 10:30 | 🔢 数学 | 具体任务描述
- [ ] 14:00 – 16:00 | 📡 专业课 | 具体任务描述
- [ ] 21:00 – 21:30 | 📝 复盘 | 填写复盘区

---

## 📊 晚间复盘

| 指标 | 计划 | 实际 |
|:----:|:----:|:----:|
| 数学用时 | Xh | |
| 数学题量 | X 题 | |
| 单词数 | 150 | |
| 专业课用时 | Xmin | |
| 完成率 | — | |

> [!note] 今日总结
> _（晚间填写）_

---

## 🔗 关联
- 上一日：[[YYYY-MM-DD 周X]]
- 下一日：[[YYYY-MM-DD 周X]]
```
