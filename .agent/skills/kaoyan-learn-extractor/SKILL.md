# /learn · 考研学习模式提取器

分析当前会话，提取值得长期保存的考研学习模式，并通过 skill-creator 创建可被系统识别的正式 skill。

## 触发时机

完成以下任意情况后运行 `/learn`：
- 整理完一章节的错题（如微分方程、无穷级数）
- 总结出一类题型的通用解题思路
- 发现高频考点与其他章节的知识关联
- 建立了可复用的 Obsidian 考研笔记组织方式

---

## Phase 1：上下文分析

回顾当前会话，识别以下类型的可复用学习模式：

### 1. 易错知识点

- 哪个知识点容易出错？（概念混淆/计算失误/条件遗漏）
- 错误的根本原因是什么？
- 正确的理解方式是什么？
- 是否属于数学一高频考点？

### 2. 解题方法与技巧

- 某类题型的通用解题思路（如凑微分、分部积分、特征方程法）
- 关键解题步骤或换元策略
- 容易忽略的边界条件或特殊情况
- 一题多解的对比总结（哪种最适合考场）

### 3. 知识点关联

- 跨章节的知识联系（如微分方程 ↔ 积分、无穷级数 ↔ 极限）
- 相似概念辨析（如收敛与一致收敛、偏导数与全微分）
- 公式之间的推导关系

### 4. 专业课模式（信号与系统）

- 信号变换的物理意义与数学推导的对应关系
- 系统性质（线性/时不变/因果/稳定）的判断方法
- 卷积/傅里叶/拉普拉斯变换的解题套路

### 5. Obsidian 笔记管理与排版模式

- 发现某种 **Callout 组合**特别适合考研场景（如错题分析用 `[!warning]` + `[!success]` 对比呈现）
- 新的 **Frontmatter 字段设计**（如追踪题目难度、知识点标签、复习次数）
- **Wikilink 组织策略**（如如何把错题文件、知识点文件、计划文件三者互联）
- 高效的**笔记模板结构**（如错题页/知识点页/复盘页的段落顺序和信息层级）
- 发现的 **Obsidian 插件用法技巧**（如 kaoyan-countdown 插件的任务格式规律）
- **图片/资源的组织惯例**（如 `images/` 目录结构、嵌入方式）

### 过滤规则（不提取）

- 单次计算粗心错误（非规律性失误）
- 已有 skill 覆盖的内容：
  - `error-review-scheduler`（错题复习调度）
  - `error-template-scaffold`（错题文件创建）
  - `kaoyan-daily-plan`（每日计划格式）
  - `weekly-review`（周复盘生成）
  - `math-problem-solver`（数学解题格式）
- 过于基础的定义或公式（教材直接可查的）

---

## Phase 2：整理 Skill 输入

将提取的模式整理为以下结构：

- **Skill 名称**：kebab-case 命名，体现科目+知识点（如 `ode-particular-solution`、`series-convergence-comparison`）
- **Skill 描述**：一句话触发条件（「遇到...类型题时」或「整理...章节错题时激活」）
- **知识场景**：具体题型或知识点
- **核心方法**：解题思路/易错点/知识关联
- **考研权重**：是否为数学一高频考点（高/中/低）
- **示例**：典型例题或易错对比（如适用）

向用户展示整理结果，**确认后**进入 Phase 3。

---

## Phase 3：调用 skill-creator 创建正式 Skill

1. 调用 `skill-creator`，传入 Phase 2 整理的内容
2. 生成正确的目录结构：
   ```
   .agent/skills/[skill-name]/
   ├── SKILL.md          # 包含 YAML frontmatter (name + description)
   └── references/       # 可选：相关例题、公式推导参考
   ```
3. 确认 SKILL.md 包含正确的 YAML frontmatter：
   ```yaml
   ---
   name: skill-name
   description: 遇到[具体场景]时激活，帮助[解决什么问题]
   ---
   ```

> **重要**：不要直接写入 `learned/` 目录。必须通过 skill-creator 创建正式 SKILL.md，否则系统无法识别。

创建后同步到 `.claude/skills/` 目录（两处同步）。

---

## Phase 4：确认

- 向用户展示创建的 skill 路径和内容摘要
- 建议在相关错题文件中添加 `[[skill-name]]` 关联引用
- 每次只提取**一个最有价值**的学习模式，保持 skill 聚焦

---

## 注意事项

- 优先提取数学一高频考点（微积分、微分方程、级数、线代行列式/特征值）
- 其次提取专业课（信号与系统）的核心分析方法
- 考虑知识点在近 10 年考研真题中的出现频率
- skill-creator 是全局插件，无需额外安装
