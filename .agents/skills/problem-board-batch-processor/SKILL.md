# problem-board-batch-processor

批量处理解题板中的数学题目。适用于用户在 Gemini 完成初步解答后，需要将多道题目系统化整理到 Obsidian 解题板的场景。

## 触发条件

- 用户说"处理解题板"、"整理今天的题"、"解题板批量处理"
- 用户说"解答这些题"、"把这些题整理一下"
- 用户在解题板粘贴多张题目图片后请求处理

## 工作流程

### Phase 1: 读取待处理题目

1. 读取 `考研数学/解题板.md` 文件
2. 定位到 `# 📥 待处理题目区` 部分
3. 提取所有题目图片链接（`![[...]]` 格式）
4. 统计题目数量，告知用户："发现 N 道待处理题目，开始逐题解答..."

### Phase 2: 逐题解答与整理

对每道题目执行以下步骤：

1. **读取图片**
   - 使用 Read tool 读取题目图片
   - 识别题目内容、条件、要求

2. **知识点识别**
   - 判断所属章节（高等数学/线性代数/概率论）
   - 识别具体知识点（如：极限与连续 - 洛必达法则）
   - 标注题型（计算题/证明题/应用题/综合题）

3. **独立解答**
   - Codex 完整推导解题过程
   - 使用 LaTeX 公式格式（`$...$` 行内，`$$...$$` 块级）
   - 确保步骤清晰、逻辑严密

4. **提取通用框架**
   - 总结该类题目的通用解法步骤
   - 标注本题的特殊性（与标准解法的差异）

5. **标注易错点**
   - 至少列出 1-2 个易错点
   - 说明为什么容易出错、如何避免

6. **生成延伸训练建议**
   - 推荐同类型题目的练习方向
   - 关联相关知识点

### Phase 3: 写入整理结果

1. 将解答内容写入 `# ✅ 已整理题目` 区域
2. 每道题使用标准模板：

```markdown
## 题目 N

**📝 题目**
![[image.png]]

**🏷️ 知识点 / 题型**
[章节] - [具体知识点] - [题型]

**📌 已知条件与目标**
[提取题目的已知和求解目标]

**🧠 通用解法框架**
1. [步骤1]
2. [步骤2]
...

**🔎 本题的特殊性**
[与标准解法的差异或特殊技巧]

**🧩 详细解答**
$$
[完整推导过程，使用 LaTeX]
$$

**⚠️ 易错点**
- [易错点1及避免方法]
- [易错点2及避免方法]

**🧪 延伸训练建议**
[同类题型练习建议]

**📥 收录到错题本？**
- [ ] 是
- [ ] 否

**掌握度**（勾选一项）：
- [ ] 🟢 完全掌握
- [ ] 🟡 需要复习
- [ ] 🔴 仍有疑问

**🤔 疑难辨析**
[如有特殊疑点，在此说明]

---
```

3. 题目之间用 `---` 分隔
4. 清空 `# 📥 待处理题目区`（保留说明文字，删除图片链接）

### Phase 4: 错题收录询问

1. 整理完成后，列出所有题目的知识点
2. 询问用户："以下题目已整理完成，哪些需要收录到错题本？"
3. 用户勾选后，调用 error-collector 插件批量收录

## 质量保证

- **独立解答**：不依赖 Gemini 的答案，Codex 完整推导
- **公式规范**：所有数学公式使用 LaTeX，确保 Obsidian 可渲染
- **结构完整**：每道题必须包含所有必填字段
- **易错点必填**：至少 1 个易错点，不能为空

## 注意事项

1. **批量处理不等于降低质量**：每道题都要认真解答，不能敷衍
2. **图片路径处理**：保持原有图片链接格式 `![[...]]`
3. **题目编号**：按顺序编号（题目 1, 题目 2, ...）
4. **待处理区清理**：整理完成后清空，避免重复处理
5. **错题收录**：只收录用户勾选的题目，不要自动全部收录
