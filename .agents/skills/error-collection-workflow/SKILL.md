# 错题收录工作流

规范从「解题板」到「错题本」的完整收录流程，确保格式一致、条目分隔清晰。

## 触发

- 用户在解题板勾选 `- [x] 收录到错题本 · <章节名>`（由 error-collector 插件自动处理）
- 用户说"整理错题格式"、"修复错题排版"
- AI 辅助解题完成后提示收录

## 错题条目标准格式

每条错题之间必须用 `---` 分隔线隔开，防止上一题答案与下一题图片视觉粘连。

```markdown
---

![[images/Pasted image YYYYMMDDHHMMSS.png]]
- **错误次数**: 1
- **错误知识点**: <章节/知识点>
- **详细错误原因**：（待填写）
- **解题关键**：<一句话概括核心思路>

> [!success] 答案
> $$\boxed{<答案公式>}$$
```

### 格式要点

1. **分隔线**：每条错题前必须有 `---`（前后各一个空行）
2. **图片嵌入**：`![[images/文件名.png]]`，图片存放在同章节的 `images/` 子目录
3. **元数据**：错误次数、知识点、原因、解题关键四项必填（原因可暂填"待填写"）
4. **答案区**：使用 `> [!success] 答案` callout 折叠，公式用 `\boxed{}` 包裹
5. **首条错题**：文件第一条错题前不需要 `---`，从第二条开始加

## 收录流程（自动）

```
解题板解题完成
  ↓
用户勾选 [x] 收录到错题本 · <章节>
  ↓
error-collector 插件自动执行：
  1. 解析解题板的题目图片、解题思路、最终答案
  2. 复制图片到 考研数学/错题/<章节>/images/
  3. 查重：图片已存在 → 错误次数 +1
  4. 新题：构建标准格式条目（含 --- 分隔线）→ 追加到错题文件
  5. 重置勾选框，显示收录状态
```

## 收录流程（手动/AI 辅助）

当需要手动收录或 AI 辅助整理时：

1. 将题目截图粘贴到对应章节的错题文件中
2. 运行 `/整理图片` 将散落图片归档到 `images/`
3. 按标准格式补全元数据
4. 确保与上一条错题之间有 `---` 分隔

## 格式修复

当发现已有错题文件格式不规范（条目间缺少分隔线）时：

1. 读取错题文件全文
2. 识别每条错题的边界（以 `![[images/` 开头的行为新条目起点）
3. 在相邻条目之间插入 `\n---\n\n`（如果还没有的话）
4. 不修改条目内容本身

## 解题板格式要求

解题板（`考研数学/解题板.md`）需包含以下结构供插件解析：

```markdown
**📝 题目**
![[题目图片.png]]

**💡 解题思路**
<解题思路文本，第一句会被提取为"解题关键">

**✅ 最终答案**
$$\boxed{<答案>}$$

---
## 📥 收录到错题本？
- [ ] 收录到错题本 · <章节名>
```

## 边界条件

- 同一道题可收录到多个章节（勾选多个 checkbox）
- 重复收录同一图片时自动累加错误次数，不创建新条目
- 错题文件不存在时自动创建（含 images/ 目录）
